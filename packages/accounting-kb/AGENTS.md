# Agent Instructions — `@workspace/accounting-kb`

Vendored machine-readable subset of the Czech accounting KB (Track B, WP-0.3). The Brain reads this as
**reference knowledge** — a confidence signal feeding the KB-rule strength, never an executable write
template. Booking stays agent-native.

## Rules

- **Never hand-edit the vendored data** (`coa.json`, `predkontace/*`, `decision-trees/*`,
  `q-pattern-index.md`) or `version.json`. They are generated by `scripts/vendor.mjs`. A manual edit
  breaks the `version.json` sha256 manifest and `verifyKbIntegrity()` fails. To change content:
  re-vendor with a bumped `--version`.
- The whitelist of vendored files lives in `scripts/vendor.mjs` (`FILES` + the decision-trees folder).
  Curating the subset = editing that list, then re-vendoring.
- Source of truth is the `accountingAfframe` Obsidian vault; this package is a snapshot. `kb_version` is
  human-set (Hleb bumps it); each eval run records it for reproducibility.
- Data files live at the package root; `src/` only holds the loader. `tsconfig` `include` is `src` only —
  the JSON is data, read at runtime via the manifest, not compiled.

## Conventions

- TypeScript 6+, ESM, source-first (`exports["."] → ./src/index.ts`).
- Tests co-located `src/**/*.test.ts` (vitest, node). `pnpm --filter @workspace/accounting-kb test`.
