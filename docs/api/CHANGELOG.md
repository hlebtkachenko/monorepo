# Changelog — Public API

> **[Concept]** Not yet served. The format below is what `api.afframe.com/docs/changelog` will publish, with RSS + JSON feeds.

This file is the source. The dev portal renders it. A future generator emits `/changelog/rss.xml` and `/changelog/feed.json`.

---

## Format

Each entry:

```markdown
## 2026-05-20 — Replace Swagger UI with Scalar at /v1/docs

**Tags**: feat
**Affects**: `/v1/docs`

`/v1/docs` now renders the Scalar API Reference. `/v1/openapi.json` and every operation are unchanged.

- Same OpenAPI 3.1 document; no spec drift.
- New CSP additions: `script-src` allows `https://cdn.jsdelivr.net`.

Refs: [AFF-220](https://linear.app/hapddev/issue/AFF-220), [API-REFERENCE.md](./API-REFERENCE.md).
```

Rules:

- One H2 per entry. `YYYY-MM-DD — short title`. Title ≤80 chars.
- One H2 anchor = one stable URL: `/docs/changelog#2026-05-20-replace-swagger-ui-with-scalar`.
- Tags one of: `feat`, `fix`, `breaking`, `deprecated`, `security`, `docs`.
- **Affects** lists routes / fields / event types impacted.
- Body: bullets, terse. Link the Linear issue and any relevant docs page.

---

## Tag semantics

| Tag          | Meaning                                                                                                    | Email?                                             |
| ------------ | ---------------------------------------------------------------------------------------------------------- | -------------------------------------------------- |
| `feat`       | New endpoint, field, event, or capability                                                                  | No (monthly digest)                                |
| `fix`        | Bug fix that doesn't change documented behaviour                                                           | No                                                 |
| `breaking`   | Behaviour change that violates the prior contract. Must be coupled to a `/vN+1` ship + Sunset announcement | YES, separate email                                |
| `deprecated` | Endpoint or field deprecation. Triggers `Deprecation` + `Sunset` headers                                   | YES, separate email                                |
| `security`   | Security fix or hardening                                                                                  | Sometimes (severity-dependent; SECURITY.md drives) |
| `docs`       | Documentation-only change                                                                                  | No                                                 |

## Feed format

### RSS — `/changelog/rss.xml`

Standard RSS 2.0. One `<item>` per entry. `<link>` is the stable per-entry URL. `<pubDate>` is the H2 date.

### JSON — `/changelog/feed.json`

[`JSON Feed 1.1`](https://www.jsonfeed.org/version/1.1/):

```json
{
  "version": "https://jsonfeed.org/version/1.1",
  "title": "Afframe Public API Changelog",
  "home_page_url": "https://api.afframe.com/docs/changelog",
  "feed_url": "https://api.afframe.com/changelog/feed.json",
  "items": [
    {
      "id": "2026-05-20-replace-swagger-ui-with-scalar",
      "url": "https://api.afframe.com/docs/changelog#2026-05-20-replace-swagger-ui-with-scalar",
      "title": "Replace Swagger UI with Scalar at /v1/docs",
      "date_published": "2026-05-20T00:00:00Z",
      "tags": ["feat"],
      "content_markdown": "..."
    }
  ]
}
```

Both feeds are generated by the dev portal at build time from this Markdown file. No runtime work.
